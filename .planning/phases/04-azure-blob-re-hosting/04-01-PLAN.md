**Workflow structure (8 nodes, linear with one error branch):**

Nodes (in order):

1. **`Execute Sub-workflow Trigger`** (type: `n8n-nodes-base.executeWorkflowTrigger`, typeVersion 1)
   - `inputSource`: `passthrough` (accepts whatever JSON the parent passes)
   - Expected input shape (document in the node notes):
     ```json
     {
       "image_urls": [{ "index": 1, "url": "https://..." }, { "index": 2, "url": "https://..." }],
       "post_id": "propulsar_1712790000000",
       "approval_number": "34612345678"
     }
     ```

2. **`🔧 Explode image_urls`** (type: `n8n-nodes-base.code`, typeVersion 2, mode: `runOnceForAllItems`)
   - jsCode:
     ```javascript
     // Fan out the incoming image_urls array into one item per image.
     // Carry post_id and approval_number on every item for downstream access.
     const input = $input.first().json;
     const urls = input.image_urls || [];
     const postId = input.post_id;
     const approvalNumber = input.approval_number;

     if (!Array.isArray(urls) || urls.length === 0) {
       throw new Error('Sub-workflow input missing image_urls array');
     }

     return urls.map((entry) => ({
       json: {
         slide_index: entry.index,
         source_url: entry.url,
         post_id: postId,
         approval_number: approvalNumber,
       }
     }));
     ```

3. **`📥 HTTP GET — Download source image`** (type: `n8n-nodes-base.httpRequest`, typeVersion 4.2)
   - Method: `GET`
   - URL: `={{ $json.source_url }}`
   - Options -> Response -> Response Format: `File`
   - Options -> Response -> Put Output File in Field: `data`
   - Options -> Retry -> enabled: true, maxTries: 3, waitBetweenTries: 2000
   - No auth, no custom headers (the Ideogram/FAL URLs already include their own `exp`/`sig` query params)
   - Notes: "REHOST-01 — download source image as binary, preserving exp/sig query params from the source URL"

4. **`🔧 Build blob name`** (type: `n8n-nodes-base.code`, typeVersion 2, mode: `runOnceForEachItem`)
   - jsCode:
     ```javascript
     // Generate date-prefixed UUID blob name.
     // Detect extension from source URL ending, default to .jpg.
     const { randomUUID } = require('crypto');

     const now = new Date();
     const y = now.getUTCFullYear();
     const m = String(now.getUTCMonth() + 1).padStart(2, '0');
     const d = String(now.getUTCDate()).padStart(2, '0');

     // Infer extension from URL path (strip query string first).
     const sourceUrl = $json.source_url || '';
     const pathOnly = sourceUrl.split('?')[0].toLowerCase();
     let ext = 'jpg';
     let contentType = 'image/jpeg';
     if (pathOnly.endsWith('.png')) {
       ext = 'png';
       contentType = 'image/png';
     } else if (pathOnly.endsWith('.webp')) {
       ext = 'webp';
       contentType = 'image/webp';
     }

     const blobName = `${y}/${m}/${d}/${randomUUID()}.${ext}`;

     return {
       json: {
         ...$json,
         blob_name: blobName,
         content_type: contentType,
       }
     };
     ```

5. **`📤 HTTP PUT — Upload to Azure Blob`** (type: `n8n-nodes-base.httpRequest`, typeVersion 4.2)
   - Method: `PUT`
   - URL: `=https://{{ $env.AZURE_STORAGE_ACCOUNT }}.blob.core.windows.net/{{ $env.AZURE_CONTAINER }}/{{ $json.blob_name }}?{{ $env.AZURE_SAS_PARAMS }}`
   - Send Headers: true, headerParameters:
     - `x-ms-blob-type`: `BlockBlob`
     - `x-ms-version`: `2020-10-02`
     - `Content-Type`: `={{ $json.content_type }}`
   - Send Body: true
   - Body Content Type: `Binary` (`contentType: "binaryData"`)
   - Input Data Field Name: `data` (must match the field set in Task 3 node)
   - Options -> Retry -> enabled: true, maxTries: 3, waitBetweenTries: 2000 (PUT is idempotent, safe to retry)
   - Options -> Response -> Full Response: true (so we can assert status 201)
   - Continue On Fail: **false** (let errors propagate to the Stop And Error node via the error output)
   - On Error: `continueErrorOutput` — this property lives at the **node object root level**, alongside `id`, `name`, `type` — NOT inside `parameters` or `parameters.options`. Exact JSON shape for this node:
     ```json
     {
       "id": "http-put-blob",
       "name": "📤 HTTP PUT — Upload to Azure Blob",
       "type": "n8n-nodes-base.httpRequest",
       "typeVersion": 4.2,
       "onError": "continueErrorOutput",
       "parameters": {
         "method": "PUT",
         "url": "=https://{{ $env.AZURE_STORAGE_ACCOUNT }}.blob.core.windows.net/{{ $env.AZURE_CONTAINER }}/{{ $json.blob_name }}?{{ $env.AZURE_SAS_PARAMS }}",
         "sendHeaders": true,
         "sendBody": true,
         "contentType": "binaryData",
         "inputDataFieldName": "data"
       }
     }
     ```
     Wire the error output (output index 1) to the `📱 Notify Abort WA` node (see connections below).
   - Notes: "REHOST-02/03 — PUT to Azure Blob with SAS. x-ms-blob-type REQUIRED. Retry safe because PUT is idempotent (unlike media_publish)."

6. **`🔧 Build blob URL`** — Use a **Set node** (NOT a Code node) so `{{ $env.AZURE_STORAGE_ACCOUNT }}` and `{{ $env.AZURE_CONTAINER }}` are used in expression fields, honoring the "no $env in Code nodes" locked decision.
   - Type: `n8n-nodes-base.set`, typeVersion 3
   - Name: `🔧 Build blob URL`
   - Mode: Manual Mapping, Keep Only Set = false (all upstream fields pass through)
   - Fields:
     - `blob_url` (string): `=https://{{ $env.AZURE_STORAGE_ACCOUNT }}.blob.core.windows.net/{{ $env.AZURE_CONTAINER }}/{{ $('🔧 Build blob name').item.json.blob_name }}`
     - `slide_index` (number): `={{ $('🔧 Build blob name').item.json.slide_index }}`
     - `post_id` (string): `={{ $('🔧 Build blob name').item.json.post_id }}`
     - `approval_number` (string): `={{ $('🔧 Build blob name').item.json.approval_number }}`
   - Notes: "REHOST-04 — permanent public URL. Env vars read via {{ $env.* }} expression (allowed), NOT via $env inside Code node (forbidden per CLAUDE.md)."

7. **`✅ HTTP HEAD — Verify public reachability`** (type: `n8n-nodes-base.httpRequest`, typeVersion 4.2)
   - Method: `HEAD`
   - URL: `={{ $json.blob_url }}`
   - No auth, no custom headers (this is the anonymous read check)
   - Options -> Retry -> enabled: false (fail fast — a 403 here means Azure is misconfigured, not a transient error)
   - Options -> Response -> Full Response: true
   - Continue On Fail: false
   - Notes: "Catches 'AllowBlobPublicAccess disabled' misconfiguration. Returns 200 if blob is publicly reachable. 403 = Felix's Azure Portal setup was not completed."

8. **`🗂️ Collect blob_urls`** (type: `n8n-nodes-base.code`, typeVersion 2, mode: `runOnceForAllItems`)
   - jsCode:
     ```javascript
     // Aggregate all per-item blob URLs into a single output item, sorted by slide_index.
     const items = $input.all();
     const postId = items[0]?.json?.post_id || null;

     const blobUrls = items
       .map(it => ({
         index: it.json.slide_index,
         url: it.json.blob_url,
       }))
       .filter(b => b.url)
       .sort((a, b) => a.index - b.index);

     return [{
       json: {
         blob_urls: blobUrls,
         post_id: postId,
       }
     }];
     ```

9. **`🛑 Abort + Notify`** (error branch) — two nodes chained:
   - **Node 9a: `📱 Notify Abort WA`** (type: `n8n-nodes-base.httpRequest`, typeVersion 4.2)
     - Method: POST
     - URL: `https://api.ycloud.com/v2/whatsapp/messages`
     - Headers: `X-API-Key: ={{ $env.YCLOUD_API_KEY }}`, `Content-Type: application/json`
     - Body (JSON): `={{ JSON.stringify({ from: $env.YCLOUD_WHATSAPP_NUMBER, to: $json.approval_number, type: "text", text: { body: "❌ Error al re-alojar imagen para el post (slide " + ($json.slide_index || '?') + "). Post cancelado. Reintenta desde el Wizard." } }) }}`
     - Notes: "Abort notification — preserves slide_index in the message so Felix knows which image failed"
   - **Node 9b: `🛑 Stop And Error`** (type: `n8n-nodes-base.stopAndError`, typeVersion 1)
     - errorMessage: `={{ "Re-host failed for slide " + ($json.slide_index || 'unknown') + " after 3 retries. Post aborted." }}`
     - Notes: "Halts the sub-workflow execution so the main workflow sees a failed execution and does NOT proceed to Meta publish."
