**Test B — Duplicate prevention (retry disabled on media_publish)** (covers Success Criterion 3, IGPUB-04, ERR-02)

This is the most important test in Phase 5. Approach: force `media_publish` to "fail" from n8n's perspective after Meta has likely processed it, then confirm IG profile has exactly 0 or 1 new post — NEVER 2.

1. In n8n-azure UI, open the main workflow in edit mode.
2. Select `🚀 IG: media_publish` node.
3. In the node's Settings tab (NOT Parameters), find "Timeout" under Options. **Temporarily** set it to `1500` (1.5 seconds). This is short enough that Meta may complete the publish but n8n times out waiting for the response.
4. Save the workflow (keep it active).
5. Run `node wizard/run.js`, produce a new single-photo brief. Use a distinct topic so you can identify the test post on IG (e.g., "TEST Phase 5 duplicate check <timestamp>").
6. Reply `SI` in WhatsApp.
7. Watch the n8n execution. One of three outcomes will occur:
   - **Outcome 1 (ideal — proves retry is disabled):** media_publish times out. The node shows the timeout error. Because `retryOnFail: false`, there is NO retry. Execution aborts. No WhatsApp success message. Open IG profile — either 0 new posts (Meta never got the request in time) or exactly 1 new post (Meta processed it but n8n didn't hear back). NEVER 2 posts.
   - **Outcome 2 (lucky timing — still proves correctness):** The request somehow completes within 1.5s. Execution continues normally. 1 new post on IG. Not a clean duplicate-prevention test, but not a failure either.
   - **Outcome 3 (BUG — fail condition):** media_publish times out, n8n somehow retries, and TWO posts appear on IG with the same caption within seconds of each other. This would indicate retry is NOT actually disabled. Phase 5 CANNOT ship.
8. Open `instagram.com/propulsar.ai` and scroll the latest 3 posts. Count the posts with the test topic/timestamp/caption:
   - [ ] Count is 0 or 1 — PASS.
   - [ ] Count is 2+ — FAIL. Phase 5 is blocked until retry config is verified correct.
9. **Restore the timeout to the default.** Edit the media_publish node, Settings tab, Timeout, remove the value (or set back to `300000` = 5 min default). Save.
10. Run Test A one more time (smoke test) to confirm restoration is clean.

**Test B is PASS iff step 8 count is 0 or 1, AND step 10 smoke-test of Test A passes after restoration.**
