  **B. Inject context before Meta calls via Merge Rehost Output**

  The `🔗 Merge Rehost Output` Set node already passes `approval_number`, `topic`, and (after Plan 01) `hashtag_block`. Add two more fields:
  - `blob_urls`: `{{ $json.blob_urls }}` — from the Re-host Images sub-workflow output. This is needed for blob cleanup in error paths.

  Additionally, modify `🔧 Prep Re-host Input` to also pass through `blob_urls` from session if available (for cases where re-host already ran before error).

  Since all Meta nodes are DOWNSTREAM of Merge Rehost Output, the error output items will carry these fields via `$json` inheritance. No per-node Set node prefix needed — the item flowing into each Meta node already has `approval_number`, `topic`, and `blob_urls` from Merge Rehost Output.

  HOWEVER: the error output from an HTTP Request node does NOT forward the input `$json` — it returns the HTTP error response body only. So we DO need to handle this differently.

  **Revised approach for context in error paths:**
  The error output item will have only the HTTP error body (Meta error JSON). To get `approval_number`, `topic`, and `blob_urls` in the error handler, the "🚨 Parse Meta Error" Code node must use cross-references:
  ```javascript
  const mergeData = $('🔗 Merge Rehost Output').first().json;
  const approval_number = mergeData.approval_number || '';
  const topic = mergeData.topic || '';
  const blobUrls = mergeData.blob_urls || [];
  ```
  This works because `🔗 Merge Rehost Output` always runs before any Meta call. Even when the Meta call fails, the Merge node's data is available via cross-ref.

  For platform identification, we need to know WHICH node errored. In n8n, when multiple nodes feed into one error handler, there's no built-in way to identify the source. Two approaches:
  - Use separate "Parse" Code nodes per path (wasteful — 10 nodes)
  - Use `$('node-name').isExecuted` checks in a single Code node (n8n may not support this)
  
  Simplest approach: Wire single-post Meta error outputs to one error handler, carousel Meta error outputs to another, and have each set the platform field explicitly.

  Actually even simpler: Use TWO error handler Code nodes:
  1. **"🚨 Parse Meta Error (Single)"** — receives errors from the 4 single-post Meta nodes. Sets `format: "single"`.
  2. **"🚨 Parse Meta Error (Carousel)"** — receives errors from the 6 carousel Meta nodes. Sets `format: "carousel"`.

  Both converge to the same downstream nodes (IF token expired → WA alert → Sheets fail log → blob cleanup).

  Wait — that still doesn't tell us WHICH specific node failed (IG vs FB). But the Meta error response body includes `fbtrace_id` which is enough for debugging. The URL pattern in the error also tells us (instagram vs facebook). So the platform detection can be done from the error URL or we just report "IG/FB" generically based on which path.

  **FINAL APPROACH — keep it simple with ONE error handler:**

  Wire all 10 Meta node error outputs to a single Code node "🚨 Parse Meta Error". This node:
  1. Extracts Meta error fields from `$json` (the HTTP error response body)
  2. Gets context via cross-ref to `🔗 Merge Rehost Output`
  3. Sets `platform_failed` based on which cross-refs are available:
     - Try `$('🔗 IG: Get Permalink').first()` — if it executed, IG was successful, so error is on FB side
     - Otherwise, error is on IG side (or both)
     - For carousel, same logic with carousel permalink node
  
  Actually, a much simpler heuristic: just report "IG/FB" based on whether the error includes a FB or IG URL pattern. But we lose that info in the error body too.

  **SIMPLEST FINAL APPROACH:** Report `platform_failed: "Meta (IG/FB)"` generically. The fbtrace_id + error_code + error_message are what matter for debugging. The NOTIF-02 requirement says "which platform failed (IG/FB/both)" but since both platforms use the same token and the error propagation is sequential (IG first, then FB), if IG fails, FB never runs. So:
  - If single-post path errors: report "IG" if error is from ig-* nodes, "FB" if from fb-* node
  - If carousel path errors: same logic
  
  To achieve this without per-node handlers, use TWO convergence points:
  - All IG error outputs → "🚨 Parse Meta Error (IG)" with `_platform = "Instagram"`
  - All FB error outputs → "🚨 Parse Meta Error (FB)" with `_platform = "Facebook"`
  
  But that's 2 Code nodes with identical logic except for a string. Better: use a single Code node and wire IG errors through a Set node that adds `_platform: "Instagram"`, and FB errors through another Set node adding `_platform: "Facebook"`.
